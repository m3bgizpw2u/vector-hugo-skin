This paragraph demonstrates how Wikipedia wraps body text around a right-floated thumbnail figure. The image itself was originally 1920 × 1080 px (a 16:9 ratio), but the thumbnail container only fixes a width (260px here). The image's height: auto rule lets the browser scale it down proportionally — so it renders at roughly 260 × 146px — with no cropping and no distortion. This is the whole trick: width is fixed, height follows the source ratio automatically.

This second figure is left-floated instead, just like Wikipedia sometimes alternates image placement for visual rhythm down the length of a long article. Both figures use the exact same container width (260px) even though the underlying images are identical 1920×1080 sources — the layout system doesn't care about the original pixel dimensions, only about the ratio, which it preserves automatically via height: auto.

If you want images to appear larger in the mockup (Wikipedia's modern default is closer to 220–320px, but user preferences can push it up to 400px+), just change the single .thumb { max-width: ... } value — everything else recalculates on its own.

Responsiveness: resize this browser window to see it in action. Above 1024px the figures cap at 260px and float beside the text, like desktop Wikipedia. Between 641–1024px (tablet) they shrink to a 200px cap. At 640px and below (mobile) they drop the float entirely, go full-width, and stack above/below the paragraph instead of squeezing next to it — matching how Wikipedia's mobile skin behaves. The image inside never needs its own breakpoint, since width:100%; height:auto already tracks whatever width the container currently has.
